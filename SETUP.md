# 동문회 온라인 마라톤 리더보드 — 설정 가이드

전체 흐름:

```
[구글 폼] → [응답 시트] → (웹에 CSV로 게시) → [github.io 페이지가 읽어 자동 집계·정렬·표시]
```

페이지(`index.html`)가 **폼 원본 응답을 직접 읽어** 3개 트랙을 집계합니다.
시트에 수식을 만들 필요가 없어요. 폼만 받으면 새로고침 시 자동 갱신됩니다.

- **5km** — 5km 트랙 응답 중 사람별 최고 페이스, 페이스 빠른 순
- **10km** — 10km 트랙 응답 중 사람별 최고 페이스, 페이스 빠른 순
- **누적 거리** — 사람별 모든 응답의 거리 합, 많은 순

---

## 1. 구글 폼 만들기

[forms.google.com](https://forms.google.com) → 빈 양식. 아래 질문을 만듭니다.
(질문 제목에 아래 **키워드**만 들어가면 페이지가 자동 인식합니다. 문구는 바꿔도 됨)

| # | 질문 제목 (키워드)      | 유형        | 필수 | 설명 |
|---|------------------------|------------|------|------|
| 1 | **학번**               | 단답형      | ✔   | 예: 08 |
| 2 | **이름**               | 단답형      | ✔   | 실명 |
| 3 | **트랙**               | 객관식      | ✔   | 보기: `5km`, `10km` |
| 4 | **거리** (km)          | 단답형(숫자) | ✔   | 이번에 달린 거리. 예: 5, 10, 7.5 |
| 5 | **기록**               | 단답형      | ✔   | 완주 시간. 예: `25:30`, `1:02:48` (페이스 `5:30/km`도 인식) |
| 6 | **기록 사진**          | 파일 업로드  | ✔   | 러닝 앱 캡처. *관리자 검증용* |

> **참가자는 매번(달릴 때마다) 폼을 제출**합니다. 누적 거리는 제출들의 거리 합으로 자동 계산돼요.

### 팁
- **기록**은 완주 "시간"으로 받는 걸 권장합니다(페이스는 페이지가 자동 계산). 페이스를 직접 받고 싶으면 `5:30/km`처럼 받으세요.
- 파일 업로드 질문을 쓰면 응답자는 **구글 로그인**이 필요합니다(동문/재학생이면 보통 문제 없음). 업로드된 사진은 관리자 드라이브에 저장됩니다.
- 설정 → 응답 → "이메일 주소 수집" 켜두면 중복/문의 대응이 쉽습니다.

## 2. 응답 시트 연결 & CSV로 게시

1. 폼 상단 **응답** 탭 → 초록색 시트 아이콘 → **Google Sheets에 연결**.
2. 열린 시트에서 메뉴 **파일 → 공유 → 웹에 게시**(Publish to web).
3. 대화상자에서
   - **게시 대상**: `설문지 응답 시트1` (전체 문서 아님)
   - **형식**: `쉼표로 구분된 값(.csv)`
   - **게시** 클릭 → 나오는 링크 복사. 형태:
     ```
     https://docs.google.com/spreadsheets/d/e/XXXX/pub?gid=0&single=true&output=csv
     ```
4. (선택) 시트에 **검증** 열을 직접 추가할 수 있습니다. 관리자가 사진과 기록을 대조한 뒤 `✓`(또는 `y`)를 적으면 페이지에 ✓ 배지가 표시됩니다. 헤더 이름에 "검증"이 들어가면 됩니다.

> 보안 메모: "웹에 게시"는 **그 시트 내용**만 CSV로 공개합니다. 사진은 드라이브에 따로 저장되어 공개 페이지에 노출되지 않습니다(관리자 검증용).

## 3. 페이지에 시트 URL 넣기

`index.html` 상단 `CONFIG` 블록만 수정합니다.

```js
const CONFIG = {
  SHEET_CSV_URL: "여기에_2번에서_복사한_CSV_링크",
  eventName: "첨단융합공학부 동문회 온라인 마라톤",
  eventMeta: "2026 Summer",
  notice: "공지 문구...",
  onlyVerified: false,   // true 면 '검증됨' 기록만 순위에 반영
};
```

저장 후 파일을 브라우저로 열어 실제 데이터가 뜨는지 확인하세요.
(비워두면 샘플 데이터가 표시됩니다.)

---

## 4. GitHub Pages 배포

리포지토리에 `index.html`과 `uploads/` 폴더를 올리고 Pages를 켜면 됩니다.
자세한 명령은 같이 진행합니다. 배포 후 주소:

```
https://<사용자명>.github.io/<리포명>/
```

### 갱신 방식
- 참가자가 폼을 제출 → 시트 자동 반영 → **페이지 새로고침 시 자동 최신화**.
- 코드/디자인을 바꾼 경우에만 다시 `git push` 하면 됩니다.
