# 동문회 온라인 마라톤 리더보드 — 설정 가이드

전체 흐름:

```
[구글 폼] → [응답 시트(비공개 유지)] → [Apps Script 웹앱이 필요한 열만 JSON으로 반환] → [github.io 페이지가 읽어 자동 집계·정렬·표시]
```

핵심: **시트를 공개하지 않습니다.** Apps Script가 학번·이름·트랙·거리·기록·검증여부만 추려서 반환하고,
사진·이메일·타임스탬프 등 나머지 열은 노출되지 않습니다. 폼만 받으면 새로고침 시 자동 갱신됩니다.

페이지(`index.html`)의 3개 탭:
- **5km** — 5km 트랙 기록 중 사람별 최고 페이스, 빠른 순
- **10km** — 10km 트랙 기록 중 사람별 최고 페이스, 빠른 순
- **누적 거리** — 사람별(학번+이름) 모든 거리 합, 많은 순

---

## 1. 구글 폼 만들기

[forms.google.com](https://forms.google.com) → 빈 양식. 아래 질문을 만듭니다.
질문 제목에 아래 **키워드**가 포함되면 페이지가 자동 인식합니다(영문/한글 모두 OK, 순서 무관).

| # | 질문 제목 (키워드)             | 유형        | 필수 | 설명 |
|---|------------------------------|------------|------|------|
| 1 | **학번**                     | 단답형      | ✔   | 예: 15 |
| 2 | **이름**(또는 성함)           | 단답형      | ✔   | 실명 |
| 3 | **트랙**(또는 종목·코스·Track) | 객관식      | ✔   | 보기: `5km`, `10km` |
| 4 | **거리**(총 거리)             | 단답형(숫자) | ✔   | 실제로 달린 거리. 예: 5, 10, 6 |
| 5 | **기록**(또는 시간·페이스·pace·Face) | 단답형  | ✔   | 아래 ⚠️ 주의 |
| 6 | **기록 사진**                | 파일 업로드  | ✔   | 러닝 앱 캡처. *관리자 검증용, 공개 안 됨* |

> **참가자는 매번(달릴 때마다) 폼을 제출**합니다. 누적 거리는 제출들의 거리 합으로 자동 계산됩니다.

### ⚠️ 기록(5번) 입력 방식 — 페이지 설정과 반드시 일치시킬 것
기록 열에 무엇을 받느냐에 따라 `index.html`의 `recordIsPace` 값을 맞춰야 합니다.

- **1km당 페이스로 받는 경우** (예: `5:30` = 5분 30초/km) → `recordIsPace: true`
- **완주 총 시간으로 받는 경우** (예: `25:30`, `1:02:48`) → `recordIsPace: false` (페이지가 시간÷거리로 페이스 계산)

> 현재 이 이벤트는 **페이스(`recordIsPace: true`)** 로 설정돼 있습니다.
> 잘못 맞추면 페이스가 엉뚱하게(예: 5:30 → 0:55/km) 표시됩니다.

### 팁
- 트랙(3번)은 **반드시 `5km` / `10km` 텍스트**가 들어가게 하세요(객관식 권장). 페이지가 이 값으로 탭을 나눕니다.
- 파일 업로드 질문을 쓰면 응답자는 **구글 로그인**이 필요합니다(동문/재학생이면 보통 문제 없음).
- 설정 → 응답 → "이메일 주소 수집"을 켜두면 중복/문의 대응이 쉽습니다(이메일은 페이지에 노출되지 않음).

## 2. 응답 시트 연결

1. 폼 상단 **응답** 탭 → 초록색 시트 아이콘 → **Google Sheets에 연결**.
2. (선택) 시트에 **검증** 열을 직접 추가할 수 있습니다. 관리자가 사진·기록을 대조한 뒤 `✓`(또는 `y`)를 적으면
   페이지에 검증 배지가 표시됩니다. 헤더에 "검증"·"확인"이 들어가면 됩니다.

> 이 단계에서 시트를 **공개(웹에 게시)하지 않습니다.** 시트는 비공개로 둡니다.

---

## 3. Apps Script 웹앱 배포 (시트를 JSON API로 노출)

> ⚠️ **Apps Script는 폼이 아니라 "연결된 응답 시트"에서 엽니다.**
> 폼에서 열면 시트에 접근하지 못해 "잘못된 접근" 등의 오류가 납니다.

1. **2번에서 연결한 응답 시트(Google Sheets)** 를 엽니다.
2. 메뉴 **확장 프로그램 → Apps Script**.
3. 편집기 내용을 모두 지우고 `leaderboard_api.gs` 코드 전체를 붙여넣고 저장(⌘/Ctrl+S).
4. 우측 상단 **배포 → 새 배포**
   - 유형: **웹 앱**
   - 다음 사용자로 실행: **나(스크립트 소유자)** ← 비공개 시트에 직접 접근
   - 액세스 권한: **모든 사용자** ← 페이지가 fetch 할 수 있도록
5. 처음 배포 시 **권한 승인** 화면이 뜹니다(정상):
   - REVIEW PERMISSIONS → 계정 선택 → "Google에서 확인하지 않은 앱" 경고에서
     **고급 → (프로젝트)로 이동(안전하지 않음) → 허용**
   - 본인이 만든 본인 스크립트라 안전합니다.
6. 나오는 **웹 앱 URL**을 복사합니다. 반드시 **`/exec`로 끝나야 합니다**(`/dev`는 테스트용이라 안 됨).
   ```
   https://script.google.com/macros/s/XXXXXXXX/exec
   ```

### "잘못된 접근입니다"가 뜨면 (멀티 로그인 버그)
구글 계정을 여러 개 로그인해 두면 자주 발생합니다.
- 시크릿 창에서 시트 소유 계정 **하나만** 로그인 후 다시 시도, 또는
- 주소창 URL의 `/u/숫자/` 부분을 `/u/0/`(기본 계정)으로 바꿔 접속.

### 코드를 고친 뒤에는 — 꼭 "새 버전"으로 재배포
Apps Script는 **저장만으로는 `/exec`에 반영되지 않습니다.**
> 배포 → **배포 관리** → 연필(편집) → 버전 **"새 버전"** → 배포  *(URL은 그대로 유지됨)*

## 4. 페이지에 설정 넣기

`index.html` 상단 `CONFIG` 블록만 수정합니다.

```js
const CONFIG = {
  APPS_SCRIPT_URL: "여기에_3번에서_복사한_/exec_URL",
  eventName: "첨단융합공학부 동문회 온라인 마라톤",
  eventMeta: "2026 Summer",
  notice: "공지 문구...",
  onlyVerified: false,   // true 면 '검증됨' 기록만 순위에 반영
  recordIsPace: true,    // 폼 기록 열이 'km당 페이스'면 true, '총 완주 시간'이면 false (1번 ⚠️ 참고)
};
```

확인 방법: `APPS_SCRIPT_URL`을 브라우저 주소창에 직접 쳐서 `{"records":[...]}` JSON이 나오면 정상입니다.
- `"track":""` 로 비어 있으면 → 시트 트랙 열 제목/값 또는 `.gs` 재배포 확인
- JSON은 정상인데 페이지가 비면 → 강력 새로고침(⌘+Shift+R)

## 5. GitHub Pages 배포

이 리포지토리는 `sitalumni.github.io` (사용자/조직 페이지)라 `main`에 push하면 바로 배포됩니다.

```
https://sitalumni.github.io/
```

### 갱신 방식
- **참가자가 폼 제출 → 시트 자동 반영 → 페이지 새로고침 시 자동 최신화** (push 불필요).
- **코드/디자인을 바꾼 경우에만** 다시 `git push`.
- **`.gs`(시트 읽는 로직)를 바꾼 경우**: github push가 아니라 **Apps Script에서 "새 버전" 재배포**해야 적용됩니다(3번 참고). 리포의 `.gs`는 사본일 뿐 실행되지 않습니다.
